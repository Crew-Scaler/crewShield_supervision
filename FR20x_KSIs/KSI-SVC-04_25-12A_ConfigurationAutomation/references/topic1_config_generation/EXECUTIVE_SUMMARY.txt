KEY FINDINGS & TRENDS
================================================================================

1. HYBRID LLM + FORMAL METHODS (CloudFix)
   - Pure LLM generation insufficient for compliance
   - Combine LLMs with SMT solvers/static analysis
   - Verification loop: generate → verify → repair

2. CONTEXT-AWARE COMPLIANCE (Medical Malice)
   - Generic harm definitions fail for regulatory frameworks
   - Need domain-specific reasoning, not keyword matching
   - Reasoning-based violation detection critical

3. MODULAR BEATS END-TO-END (PentestEval)
   - End-to-end autonomous agents: 31% success rate
   - Stage-level modularization improves reliability
   - Pipeline: parse → select → generate → verify

4. TRUST & AUTHORIZATION (SoK Paper)
   - Trust-authorization mismatches in LLM agents
   - NIST framework alignment required
   - Authorization constraints must be encoded

5. EXPLAINABILITY REQUIRED (Toward Explaining LLMs)
   - Black-box outputs insufficient for compliance
   - Audit trails essential for FedRAMP/NIST
   - Reasoning transparency for regulatory acceptance

6. CONSTRAINT-DRIVEN GENERATION (LLM as Neural Architect)
   - Strict constraints before LLM invocation
   - Template-based with controlled filling
   - API contract enforcement applicable to policies
