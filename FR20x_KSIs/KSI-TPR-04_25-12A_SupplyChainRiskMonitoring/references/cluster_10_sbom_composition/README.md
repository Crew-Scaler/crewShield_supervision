## Key Research Findings

### SBOM Maturity Challenges
- Tool inconsistency between SPDX and CycloneDX
- 97.5% false positive rate in vulnerability analysis
- "SBOM confusion vulnerability" in container security
- Only 0.56% GitHub adoption of policy-driven SBOMs

### AI/ML System Transparency
- Growing importance of AIBOM, AIRS, and TAIBOM frameworks
- 3,859 LLM applications analyzed with 109,211 models
- 55,997 PyTorch repositories studied for composition patterns
- New provenance frameworks for ML artifacts

### Vulnerability Assessment Issues
- Generator-dependent variations in results
- VEX format maturity concerns
- Need for improved automated assessment tools
- Function-level analysis to reduce false alarms

### Security & Integrity
- SBOM generation vulnerable to insider threats
- Selective disclosure with cryptographic integrity
- TEE-based build verification mechanisms
- 98% attack prevention with runtime enforcement
